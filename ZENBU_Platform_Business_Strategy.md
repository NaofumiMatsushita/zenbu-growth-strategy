# ZENBU住宅トータルプラットフォーム事業戦略
## 住宅ライフサイクル全体をカバーする日本初の総合プラットフォーム

策定日：2026年1月
ZENBU株式会社 新規事業開発部

---

## エグゼクティブサマリー

### 事業コンセプト
**「ZENBU LIFE（ゼンブライフ）」**
入居前のマッチングから入居中のサポート、退去後まで、住宅に関わる全てのステークホルダーをつなぐ日本初の住宅ライフサイクル統合プラットフォーム

### 市場機会
- **賃貸住宅市場**: 約1,900万戸、管理市場3.5兆円
- **既存プレイヤーの課題**:
  - ウチコミ等のマッチングサイト：入居後のサポートが手薄
  - 管理会社：人手不足、DX遅れ、コスト圧力
  - 入居者：トラブル時の不安、アフターサービスの不満

### ZENBUの圧倒的優位性
1. **既存4事業の実績とノウハウ** - 24時間対応体制、職人ネットワーク
2. **入居後サポートでNo.1** - 他社が手薄なアフターサービス領域を完全カバー
3. **DX推進力** - kintone等で実証済みの技術力

### 5年後の目標
- **プラットフォーム登録物件**: 50万戸
- **プラットフォーム売上**: 50億円（全社売上の30%）
- **登録大家**: 10,000名
- **登録入居者**: 100,000名
- **登録職人**: 5,000名

## 1. 市場分析と事業機会

### 1.1 既存プレイヤーの限界

| プレイヤー | 提供価値 | 限界・課題 | ZENBUの機会 |
|-----------|---------|-----------|------------|
| **ウチコミ等マッチングサイト** | 入居前のマッチング、仲介手数料無料 | 入居後のサポートなし、継続的な収益モデルが弱い | 入居後のアフターサービスで差別化 |
| **SUUMO/HOME'S** | 大量の物件情報 | 仲介会社依存、アフターサポートなし | 大家直接 + アフターサービス統合 |

## 2. 事業戦略：3つの差別化軸

### 2.1 差別化軸1: 住宅ライフサイクル全体をカバー

**競合との決定的な違い**

```
【ウチコミ等の既存プレイヤー】
入居前（物件探し） ━━━━━ マッチング ━━━━━ 契約
                                            ↓
                                        【終わり】

【ZENBU LIFE】
入居前 ━━━ マッチング ━━━ 契約 ━━━ 入居中サポート ━━━ メンテナンス ━━━ トラブル対応 ━━━ 退去サポート
  ↑                                                                                   ↓
  └─────────────────────────── ライフサイクル全体 ──────────────────────────┘
```

**提供する統合サービス**

| フェーズ | サービス内容 | ZENBUの強み |
|---------|------------|-----------|
| **入居前** | 物件マッチング、バーチャル内見、契約サポート | 既存4事業で管理している物件優先掲載 |
| **入居時** | 鍵受け渡し、ライフライン手続き代行、引越しサポート | 24時間対応、ワンストップ |
| **入居中** | 24時間トラブル対応、定期メンテナンス、設備点検 | **既存事業の強み最大活用** |
| **退去時** | 退去立会い、原状回復、次の物件紹介 | 職人ネットワークで迅速対応 |

### 2.2 差別化軸2: サブスク型アフターサービス

**「ZENBU安心パック」（月額制）**

入居者向けサブスクリプションサービス

| プラン | 月額 | サービス内容 |

- **年間収益**: 30,000名 × 1,500円 × 12ヶ月 = **5.4億円**

### 2.3 差別化軸3: データ活用による予測型サービス

**AIによる住宅メンテナンス最適化**

1. **設備劣化予測**
   - IoTセンサー + 過去データ
   - 「そろそろエアコンの点検時期です」と自動通知
   - トラブル発生前に予防

2. **職人マッチング最適化**
   - AIが最適な職人を自動選定
   - GPS + スキル + 評価 + 稼働状況
   - 到着時間を最小化

3. **大家向け資産価値最大化提案**
   - データ分析による最適な投資提案
   - 「この設備を更新すると入居率が15%向上します」

## 4. プロダクト戦略

### 4.1 MVPの定義（Year 1）

**必須機能（Phase 1）**

1. **物件検索・掲載機能**
   - 大家による物件登録
   - 入居者による物件検索・お気に入り
   - メッセージ機能（大家 ⇄ 入居者）

2. **入居者向けアプリ（iOS/Android）**
   - トラブル報告機能
   - チャットサポート
   - メンテナンス履歴確認

3. **職人マッチングシステム**
   - 案件自動配信
   - GPS連携による最適職人選定
   - 評価・レビュー機能

4. **管理画面（Web）**
   - 大家向けダッシュボード
   - 管理会社向け業務管理
   - 職人向け案件管理

**段階的追加機能（Phase 2-3）**

- バーチャル内見（VR/360度）
- AI チャットボット
- IoT センサー連携
- サブスク決済システム
- データ分析ダッシュボード

### 4.2 技術スタック

| レイヤー | 技術選定 | 理由 |
|---------|---------|------|
| **フロントエンド（Web）** | React + TypeScript | モダン、保守性高い |
| **フロントエンド（Mobile）** | React Native | iOS/Android同時開発 |
| **バックエンド** | Node.js + Express / Python（AI部分） | スケーラビリティ |
| **データベース** | PostgreSQL + Redis | 信頼性、キャッシュ |
| **インフラ** | AWS（EC2, RDS, S3, Lambda） | スケーラブル、実績 |
| **AI/ML** | TensorFlow / Scikit-learn | 予測モデル構築 |
| **地図・GPS** | Google Maps API | 職人マッチング |
| **決済** | Stripe | サブスク対応 |

## 8. リスク分析と対策

### 8.1 主要リスク

| リスク分類 | 具体的リスク | 対策 |
|----------|------------|------|
| **競合リスク** | SUUMO等大手の参入 | • 早期にニッチ（アフターサービス）で確立<br>• データ・ネットワーク効果で参入障壁構築<br>• 特許・商標取得 |
| **技術リスク** | システム開発の遅延 | • 経験豊富なPM・CTOの採用<br>• アジャイル開発<br>• 外部ベンダー活用 |
| **ユーザー獲得リスク** | 大家・入居者の獲得困難 | • 既存顧客基盤活用<br>• インセンティブ設計（初期無料等）<br>• 段階的拡大 |
| **規制リスク** | 不動産業法等の規制変更 | • 弁護士・コンプライアンス体制<br>• 業界団体との連携<br>• 柔軟な事業モデル |
| **財務リスク** | 資金ショート | • 段階的な資金調達<br>• コスト管理の徹底<br>• マイルストーン達成後に次投資 |

### 8.2 成功の鍵（KSF）

1. **既存事業とのシナジー最大化**
   - 4事業の顧客基盤を最大活用
   - 職人ネットワークをプラットフォームに統合
   - クロスセル・アップセル

2. **ネットワーク効果の早期構築**
   - 大家が増える → 物件が増える → 入居者が増える → 大家が増える
   - 臨界点（クリティカルマス）の早期突破

3. **データドリブン経営**
   - ユーザー行動データの徹底分析
   - A/Bテスト文化の浸透
   - データに基づく意思決定

4. **顧客体験の徹底追求**
   - NPS（Net Promoter Score）を最重要KPIに
   - カスタマーサポートへの投資
   - 継続的なUI/UX改善

### 9.1 参入障壁の構築戦略

**4つの堀（Moat）**
