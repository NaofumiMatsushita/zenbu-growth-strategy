## 1. プロジェクト概要

### 1.1 プロジェクト名

**AI騒音仲裁支援サービス（仮称：ZENBU Mediator）**

### 1.2 プロダクト概要

賃貸住宅等における騒音トラブルの当事者間調停をAIが支援するプラットフォーム。科学的データと法的知識を基に、公平・中立な立場から双方の合意形成をサポートする。

### 1.3 プロジェクトの範囲

**対象範囲（In Scope）:**
- 騒音測定データの収集・分析
- AI による客観的評価・診断
- 仲裁プロセスのデジタル化
- 当事者間コミュニケーション支援
- 合意書類の自動生成
- 管理会社・弁護士との連携

**対象外（Out of Scope）:**
- 強制執行や法的判断（弁護士業務）
- 物理的な防音工事（紹介のみ）
- 刑事事件への介入
- 海外物件（Phase 1では日本国内のみ）

### 1.4 プロジェクトゴール

#### 短期ゴール（6ヶ月）
- MVP開発・リリース
- β版ユーザー100組の調停成功
- 合意形成率60%以上

#### 中期ゴール（1年）
- 月間500件の調停案件処理
- 合意形成率75%以上
- B2B契約20社獲得（管理会社）

#### 長期ゴール（3年）
- 月間3,000件の調停案件処理
- 騒音トラブル解決のデファクトスタンダード化
- AI仲裁の社会的認知度50%以上

## 2. 背景と目的

### 2.1 背景

#### 社会課題
- 日本国内で年間約18万件の騒音トラブルが発生
- 当事者間の感情対立により解決が困難化
- 管理会社のリソース不足で対応が後手に
- 弁護士・裁判所は時間とコストがかかる

#### 市場機会
- 既存の解決手段は「測定のみ」or「訴訟」の両極端
- 中間的な「仲裁・調停」サービスが不在
- AI活用で低コスト・迅速な解決が可能

#### 技術的背景
- GPT-4等の大規模言語モデルの成熟
- 音声解析・騒音測定技術の向上
- オンライン仲裁プラットフォームの事例増加

### 2.2 目的

#### ビジネス目的
1. 騒音トラブルの早期解決による管理会社の業務効率化
2. 入居者満足度向上による退去率低下
3. ZENBU社の新規収益源確立

#### 社会的目的
1. 騒音トラブルによるストレス・健康被害の軽減
2. 訴訟に至る前の平和的解決の促進
3. 住環境の質的向上

#### 技術的目的
1. AI仲裁技術の確立
2. 騒音データベースの蓄積と活用
3. オンライン調停プラットフォームのスタンダード化

### 4.2 詳細機能要件

#### F001: ユーザー登録・認証

**説明:**
被害者、加害者、管理会社の3種類のユーザーが登録・ログインできる。

**機能詳細:**
- メールアドレス + パスワード認証
- SMS認証（本人確認）
- ソーシャルログイン（Google、LINE）
- パスワードリセット機能
- 2段階認証（オプション）

**入力:**
- メールアドレス（必須）
- パスワード（8文字以上、英数記号）
- 氏名（必須）
- 電話番号（必須、SMS認証用）
- ユーザー種別（被害者/加害者/管理会社）
- 物件情報（住所、部屋番号）

**出力:**
- ユーザーID（UUID）
- 認証トークン（JWT）
- ウェルカムメール

**検証ルール:**
- メールアドレスの重複不可
- 電話番号の形式チェック
- SMS認証コードの有効期限（10分）

#### F002: 案件作成

**説明:**
騒音トラブルの案件を新規登録する。

**機能詳細:**
- 被害者または管理会社が作成可能
- 基本情報の入力
- 相手方の特定（任意）
- 緊急度の設定

**入力:**
- 案件タイトル（必須）
- 騒音の種類（足音/楽器/話し声/その他）
- 発生時間帯（曜日・時間）
- 継続期間（いつから）
- 詳細説明（テキスト、最大1,000文字）
- 証拠（写真・動画・音声、最大5ファイル）
- 相手方情報（部屋番号、氏名 - 任意）
- 緊急度（低/中/高）

**出力:**
- 案件ID（UUID）
- 作成日時
- ステータス（新規）

**ビジネスルール:**
- 1ユーザーあたり同時進行案件は5件まで
- 緊急度「高」は24時間以内に測定予約必須

#### F003: 騒音測定予約

**説明:**
測定日時・場所を予約する。AIスケジューラーが最適な日時を提案。

**機能詳細:**
- カレンダーUI
- 空き時間の自動表示
- 複数候補日の提示
- リマインダー設定

**入力:**
- 希望日（第1〜第3希望）
- 希望時間帯（午前/午後/夕方/夜間）
- 測定時間（1時間/2時間/24時間）
- 立ち会い希望（あり/なし）
- 特記事項（テキスト、最大500文字）

**出力:**
- 予約ID
- 確定日時
- 担当作業員情報
- 到着予定時刻
- 確認メール

**ビジネスルール:**
- 予約は72時間前まで
- キャンセルは24時間前まで無料
- 緊急対応（当日）は+5,000円

#### F004: 測定データ収集

**説明:**
現地で騒音を測定し、データをシステムにアップロードする。

**機能詳細:**
- スマホアプリまたは専用機器
- リアルタイム測定
- GPS位置情報の記録
- 写真・動画の同時記録
- クラウド自動同期

**入力:**
- 音声データ（WAV形式）
- dB値（時系列、1秒間隔）
- GPS座標
- 測定開始・終了時刻
- 現場写真（複数枚）
- 作業員メモ（任意）

**出力:**
- 測定データファイル（S3保存）
- サムネイル画像
- メタデータ（JSON）

**検証ルール:**
- ファイルサイズ上限: 500MB/件
- 測定時間の妥当性チェック（5分〜24時間）
- GPS座標が物件住所と一致するか

#### F005: AI分析・診断

**説明:**
測定データをAIが分析し、騒音レベル・音源・法的評価を行う。

**機能詳細:**
- 統計処理（平均/最大/最小dB）
- FFT解析による周波数分析
- CNNによる音源推定
- 環境基準との比較
- 受忍限度の評価
- 判例検索

**処理フロー:**
```
測定データ
  ↓
前処理（ノイズ除去）
  ↓
統計計算
  ↓
音源推定AI（CNN）
  ↓
法的評価（GPT-4）
  ↓
結果JSON生成
```

**出力:**
```json
{
  "measurement_id": "uuid",
  "statistics": {
    "average_db": 52.3,
    "max_db": 68.5,
    "min_db": 38.2,
    "std_dev": 8.7
  },
  "sound_sources": [
    {"type": "footsteps", "confidence": 0.85},
    {"type": "voice", "confidence": 0.10}
  ],
  "environmental_standard": {
    "standard_db": 45,
    "exceeds": true,
    "excess_amount": 7.3
  },
  "legal_assessment": {
    "tolerance_exceeded": "likely",
    "confidence": 0.75,
    "precedents": [...]
  },
  "recommendations": [...]
}
```

#### F006: レポート自動生成

**説明:**
分析結果をもとに、PDF形式のレポートを自動生成する。

**機能詳細:**
- GPT-4による文章生成
- グラフ・チャート作成（Chart.js）
- テンプレートエンジン（Handlebars）
- Puppeteerによる PDF 化

**レポート構成:**
1. 表紙（案件番号、日付）
2. エグゼクティブサマリー（AI生成）
3. 測定概要（日時、場所、時間）
4. 測定結果（表・グラフ）
5. 音源推定（AI分析）
6. 環境基準との比較
7. 法的評価
8. 推奨対策
9. 参考資料（判例等）

**出力フォーマット:**
- PDF（A4、10-15ページ）
- 被害者用（詳細版）
- 加害者用（簡易版）
- 管理会社用（中立版）

#### F007: 双方への通知

**説明:**
測定結果を被害者・加害者の双方に通知する。

**機能詳細:**
- メール送信
- アプリ内プッシュ通知
- SMS（オプション）
- 既読確認

**通知内容:**
- 測定完了の連絡
- レポートダウンロードリンク
- 次のステップの案内
- 回答期限（7日間）

**タイミング:**
- 測定完了後2時間以内
- 未読の場合、24時間後にリマインド

#### F008: AI仲裁提案

**説明:**
測定結果をもとに、AIが公平な解決案を提案する。

**機能詳細:**
- GPT-4による多角的分析
- 3パターンの解決案生成
- 過去の成功事例からの学習
- 双方の受容可能性を予測

**解決案の例:**
```
【パターンA: 改善重視】
1. 加害者は防音マット（厚さ5mm以上）を設置
2. 深夜23時以降は静かに歩行
3. 被害者は1ヶ月様子見
4. 改善されない場合、管理会社立ち会いで再測定

【パターンB: 補償重視】
1. 加害者は被害者に迷惑料として2万円支払い
2. 防音マット設置（費用は加害者負担）
3. 今後の注意を約束

【パターンC: 住み分け重視】
1. 加害者の部屋移動を管理会社が斡旋
2. 移動費用は双方と管理会社で1/3ずつ負担
3. 被害者は合意書に署名
```

**評価指標:**
- 実現可能性（技術的・経済的）
- 公平性（双方の負担バランス）
- 再発防止効果
- 受容確率（過去データから予測）

#### F009: オンライン協議室

**説明:**
被害者・加害者がチャット形式で話し合いを行う。AIモデレーターが仲介。

**機能詳細:**
- リアルタイムチャット
- AIモデレーター（自動応答）
- 感情分析（炎上検知）
- 不適切発言のフィルタリング
- 管理会社の閲覧（オプション）

**UIコンポーネント:**
- メッセージ入力欄
- 過去メッセージ一覧
- AI提案の表示
- 「合意」「保留」「拒否」ボタン
- 添付ファイル機能

**AIモデレーターの役割:**
```
ユーザーA: 「夜中にうるさすぎます！」

AI: 「お気持ちお察しします。具体的には何時頃、
     どのような音が気になりますか？」

ユーザーB: 「普通に歩いているだけです」

AI: 「測定結果では、深夜0時台に平均58dBの足音が
     記録されています。環境基準（45dB）を13dB
     超過しています。Bさん、防音マットの設置を
     検討いただけますか？」
```

**安全機能:**
- 暴言・誹謗中傷の自動検出
- ブロック機能
- 通報機能
- 管理者介入

**説明:**
協議で合意した内容を法的拘束力のある合意書として生成する。

**機能詳細:**

**合意書の構成:**

2. 当事者情報
3. 経緯の要約
4. 測定結果の概要
5. 合意事項（箇条書き）
6. 履行期限
7. 違反時の措置
8. 署名欄

**出力フォーマット:**
- PDF（A4、2-3ページ）
- Word（編集可能）

**サンプル:**
```
騒音トラブル解決に関する合意書

当事者:
甲: 田中太郎（302号室）
乙: 佐藤花子（402号室）
立会人: 管理会社 株式会社〇〇

甲乙は、2026年1月X日に発生した騒音トラブルに
ついて、以下の通り合意する。

2026年1月25日に実施された騒音測定の結果、
深夜時間帯に環境基準を超過する騒音が確認された。

第2条（乙の対応）
乙は、以下の対策を2026年2月10日までに実施する。

第3条（甲の対応）

第5条（違約）

甲乙各自署名押印の上、各1通を保有する。

2026年2月1日

甲: 田中太郎  印
乙: 佐藤花子  印
立会人: 〇〇  印
```

**説明:**
合意書に電子署名を行い、法的効力を持たせる。

**機能詳細:**
- SMS認証による本人確認
- タイムスタンプ
- 電子証明書の発行
- ブロックチェーン記録（オプション）

**署名フロー:**
```
1. 合意書のPDF生成
2. 署名依頼通知（メール・SMS）
3. ユーザーがログイン
4. 内容確認
5. SMS認証コード入力
6. 署名完了
7. 全員の署名完了で確定
```

**法的要件:**
- 電子署名法準拠
- タイムスタンプの付与
- 改ざん防止（ハッシュ値記録）
